1. SI & MERISE

SIEnsemble organisé de ressources pour collecter, stocker, traiter et distribuer l'information
MERISEMéthode d'analyse et conception de SI (France, 1978)
3 principes① Séparation Données / Traitements ② Démarche par niveaux ③ Démarche par étapes
3 niveauxConceptuel (QUOI) → Logique/Org. (QUI/OÙ) → Physique (COMMENT)
ModèlesDonnées : MCD → MLD → MPD | Traitements : MCT → MOT → MPT

2. MCD — Modèle Conceptuel de Données

PropriétéDonnée élémentaire atomique (plus petite unité)
EntitéObjet du SI ayant une existence propre (rectangle)
IdentifiantPropriété unique distinguant chaque occurrence (souligné)
AssociationLien entre entités, peut porter des propriétés (ovale)
Cardinalités(min,max) — nombre de participations d'une occurrence à une association

Cardinalités

0,1 Optionnel, unique 1,1 Obligatoire, unique 0,n Optionnel, multiple 1,n Obligatoire, multiple

3. MLD — Passage MCD → MLD

RègleConditionAction
R1Cardinalité max 1 — nFK (PK du côté n) → table côté 1
R2Cardinalité max n — nNouvelle table d'association (PK = FK1 + FK2)
R3Cardinalité 1,1 — 1,1Fusion ou FK dans la table dépendante
 
Notation : TABLE (PK, Attribut, #FK)
PK = souligné | FK = précédé de #

4. Extensions du MCD

Id. relativeEntité faible (rectangle double) dont la PK inclut celle du parent
Réflexive [n,n]Table d'association avec 2 FK vers la même table
Réflexive [1,n]FK auto-référentielle dans la même table
Pseudo-entitéEncapsulation d'une association pour la lier à une autre entité

5. Normalisation

FormeRègleÉlimine
1FNAttributs atomiques + PKGroupes répétitifs
2FN1FN + pas de dépendance partielleDépendance d'une partie de PK
3FN2FN + pas de dépendance transitiveDépendance entre non-clés
Moyen mnémo : 1FN = atomique | 2FN = dépend de TOUTE la clé | 3FN = dépend SEULEMENT de la clé

6. Modélisation des Flux

Acteur interneDans l'organisation (service, département)
Acteur externeHors organisation (client, fournisseur)
FluxInformation échangée (flèche numérotée émetteur → récepteur)
DomaineSous-ensemble logique d'activités
ProcessusEnsemble de traitements déclenchés par événement(s)

7. MCT — Modèle Conceptuel des Traitements

ÉvénementFait déclencheur (ovale) — externe, interne ou temporel
OpérationTraitement ininterruptible (rectangle 3 zones)
SynchronisationCondition logique ET/OU entre événements
RésultatÉvénement produit par l'opération
Règle d'émissionCondition de sortie : toujours, si OK, si refus…
